# UX 와이어프레임 정의서 (v1.0.0)

이 문서는 대국 화면의 레이아웃 배치와 "바로 착수" 옵션 설정 시의 상태 천이 및 피드백 흐름을 정의합니다.

---

## 1. 대국 화면 레이아웃 (Layout Specification)

화면 전체 구조는 세로 스크롤을 방지하고 한 화면에 모든 주요 제어 요소가 모바일 뷰포트 크기에 맞춰 배치되도록 Column 레이아웃으로 구성합니다.

```
+-------------------------------------------------------------+
| [상단 헤더 / 시스템 타이틀 또는 설정 메뉴 아이콘]          |
+-------------------------------------------------------------+
|                                                             |
|                      바둑판 영역 (Board)                     |
|                                                             |
|           Parent W/H 중 좁은 쪽 기준 최대 정사각형           |
|                                                             |
|                                                             |
+-------------------------------------------------------------+
|                 대국 현황 패널 (Status Panel)               |
|  +--------------------+----------------+-----------------+  |
|  | ● 흑 (사석수)      |   [ 착 수 ]    |      백 (사석수) ○ |  |
|  | Timer: 00:45       |     Button     |    Timer: 12:10 |  |
|  +--------------------+----------------+-----------------+  |
+-------------------------------------------------------------+
|                 기능 버튼 패널 (Control Panel)               |
|  [ 통과 (Pass) ] [ 무르기 (Undo) ] [ 추천수 (Best) ] [ 형세판단 (Eval) ] |
+-------------------------------------------------------------+
|                 기타 옵션 / 정보 그래프 / 형세바            |
+-------------------------------------------------------------+
```

### 1.1. 바둑판 영역 (Board Area)
- **부모 뷰(Parent Container) 측정**: `BoxWithConstraints`를 통해 사용 가능한 최대 너비(`maxWidth`)와 높이(`maxHeight`)를 계산합니다.
- **정사각형 결정**: `val boardSize = min(maxWidth, maxHeight)`
- **정렬**: 계산된 `boardSize`로 가로/세로 크기를 고정하고 화면 중앙에 배치합니다.

### 1.2. 대국 현황 패널 (Status Panel)
- **3분할 구조 (Row, weights = 1f, 1f, 1f)**:
  - **좌측 (흑진영)**: 
    - 흑돌 아이콘(`●`) 또는 라벨.
    - 흑이 획득한 백의 사석 개수 표시 (예: "사석: 3").
    - 시간 흐름 표현: 흑의 차례일 때 타이머가 활성화(폰트 볼드 처리 및 테두리/배경 강조)되며 흐르는 시간 초단위 표시.
  - **중앙 (동작 영역)**:
    - `[착수]` 버튼 (기본은 비활성화 상태, 가상 착수가 있는 경우에만 활성화 및 Accent 컬러 적용).
  - **우측 (백진영)**:
    - 백이 획득한 흑의 사석 개수 표시 (예: "사석: 5").
    - 백돌 아이콘(`○`) 또는 라벨.
    - 시간 흐름 표현: 백의 차례일 때 타이머가 활성화(폰트 볼드 처리 및 테두리/배경 강조)되며 흐르는 시간 초단위 표시.
- **차례 시각화**: 현재 턴인 진영(흑 또는 백)의 영역 배경색을 미려하게 그라데이션 하이라이트(예: Soft Gold or Blue Tint)하여 명확하게 소유권을 인지시킵니다.

### 1.3. 기능 버튼 패널 (Control Panel)
- 4개의 동작 버튼을 균등 분포(Row, weights = 1f씩)로 배치합니다.
- **영한 병기 또는 설정 연동**:
  - `Pass` / `통과`
  - `Undo` / `무르기`
  - `Best` / `추천수` (AI 추천수 보기)
  - `Eval` / `형세판단` (현재 승률 및 형세 분석)
- 버튼 클릭 시 즉각적인 시각 피드백(터치 Ripple Effect, 투명도 변화)을 지원합니다.

## 2. "바로 착수" 설정에 따른 상태 전이 (Interaction Flows)

### 2.1. 옵션 설정
- 설정(Drawer 메뉴 등)에서 `"바로 착수" (Direct Play)` 스위치 설정.
- **`isDirectPlayEnabled == true` (On)**
  - 바둑판 임의의 교차점 터치 -> 즉시 바둑판에 돌이 놓이고 엔진에 착수 전달 (기존 방식).
- **`isDirectPlayEnabled == false` (Off)**
  - 바둑판 임의의 교차점 터치 -> 해당 위치에 **흐릿한 돌(가상 착수, Alpha 0.5f)**이 렌더링됨.
  - 이 때 아직 실제 착수는 발생하지 않음. 대국 현황 패널의 `[착수]` 버튼이 비활성화에서 **활성화(Accent Color)** 상태로 변경됨.
  - 사용자가 다른 위치를 다시 터치하면 흐릿한 돌이 새로운 위치로 이동하며 기존 가상 착수 위치는 초기화됨.
  - 활성화된 `[착수]` 버튼을 누르면 실제 착수로 최종 확정되고 엔진에 전달됨. 그 후 가상 착수 상태는 제거되며 `[착수]` 버튼은 다시 비활성화됨.

### 2.2. 흐릿한 돌(Ghost Stone) 스타일 정의
- **흑돌 가상 착수**: 검은색 반투명 돌 (`Color.Black.copy(alpha = 0.5f)`)
- **백돌 가상 착수**: 흰색 반투명 돌 (`Color.White.copy(alpha = 0.6f)`)
- 미세한 점멸(Pulse) 애니메이션을 추가하여 확정 대기 중인 상태임을 동적으로 인지시킬 수 있습니다.
