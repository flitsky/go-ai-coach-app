# 새 판 시작 시 KataGo 프로세스 재시작 검증

## 변경

`startNewEngineGame()`에서 기존 `configure(profile) -> newGame(...)` 대신 다음 순서로 바꿨다.

```text
stop()
initialize(profile)
newGame(boardSize, ruleset)
```

목적은 반복 새 판에서 KataGo GTP 프로세스 내부 검색 트리/캐시가 이전 판을 재사용하지 못하게 하는 것이다.

## 검증 결과

수정 APK 설치 후 같은 설정으로 실기기 자동대국을 재실행했다.

- Black: `중급 5단계` (`B64`, `4500ms`)
- White: `빠른 초급 3단계` (`B16`, `2000ms`)

초반 16수 기준:

- B64: 8회, 평균 약 `4578.5ms`, 100ms 미만 0회
- B16: 8회, 평균 약 `426.2ms`, 100ms 미만 3회
- 전체 span: 약 40.4초

이로써 “새 판 전체가 거의 즉시 재생되는” 문제는 완화됐다.

## 남은 문제

같은 판 내부에서 B64의 깊은 탐색 직후, 다음 B16 분석이 3ms 수준으로 반환되는 구간은 여전히 남았다. 이는 같은 KataGo 프로세스가 직전 검색 트리의 자식 국면을 다음 턴에 재사용하는 현상으로 판단했다.
