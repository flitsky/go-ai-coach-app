# Go AI Coach 옵션 사용자 매뉴얼 초안

이 문서는 현재 Android POC 화면의 옵션과 버튼이 하는 일을 설명하기 위한 사용자 매뉴얼 초안이다. UI 명칭은 앱 화면의 짧은 영문 라벨을 기준으로 적고, 설명은 한글로 정리한다.

## 기본 원칙

- 옵션은 대국 규칙이나 엔진 상태 자체를 바꾸는 설정과, 화면에 정보를 더 보여주는 표시 옵션으로 나뉜다.
- 표시 옵션은 기본적으로 꺼둘 수 있어야 하며, 모두 꺼도 일반 대국 화면은 계속 동작해야 한다.
- `Menu` 안의 옵션은 자주 바꾸지 않아도 되는 보조 표시 설정과 엔진 설정이다.
- 플레이 화면에 바로 보이는 옵션은 대국 중 자주 켜고 끌 가능성이 높은 설정이다.

## 플레이 화면 즉시 컨트롤

### Mode

- `AI`: 흑은 사용자, 백은 KataGo 또는 stub 엔진이 두는 AI 대국 모드다.
- `2P`: 흑과 백을 모두 사용자가 두는 로컬 테스트 모드다.
- 사석 제거, 패스, 계가, 보드 상태 재현을 사람이 직접 검증할 때는 `2P`가 유용하다.

### Quick display

플레이 화면에 바로 보이는 2열 표시 옵션이다.

- `Score graph`
  - 수순별 판세 변화를 작은 그래프로 보여준다.
  - 내부 데이터는 White 기준 score lead와 White 기준 win rate를 유지한다.
  - 화면 그래프는 대국자가 직관적으로 읽기 쉽도록 위쪽을 흑 우세, 가운데를 비슷함, 아래쪽을 백 우세로 표시한다.
  - 오른쪽 축의 `B`는 흑 우세 방향, `W`는 백 우세 방향을 뜻한다.
  - AI 대국에서는 KataGo score estimate를 기록하고, 2P 모드에서는 로컬 area estimate를 기록한다.
- `Game strip`
  - 현재 차례, 수순 수, 양측 포획 수, 마지막 수를 별도 요약 줄로 보여준다.
  - 디버그 로그를 보지 않고도 대국 흐름을 빠르게 확인할 때 사용한다.
  - 기본값은 켜짐이다.

### 대국 액션 버튼

- `Pass`: 현재 차례 플레이어가 패스한다. 양 진영이 연속 패스하면 최종 계가 흐름으로 들어간다.
- `Undo`: 마지막 착수를 되돌린다. AI 대국에서는 사람 착수와 AI 응수를 한 턴 단위로 되돌린다.
- `Top Moves`: 현재 차례에 착수 가능한 지점을 최대한 많이 분석하고, 준비된 후보 위치를 보드 위에 표시한다.
  - 사람 차례가 오면 백그라운드에서 현재 합법 착점 수만큼 후보 분석을 먼저 요청한다.
  - 버튼은 토글로 동작한다. 켜진 상태에서는 매 사용자 턴마다 준비된 pre-move analysis cache를 보드에 자동 표시한다.
  - 꺼진 상태에서도 착수 평가를 위한 pre-move analysis cache는 백그라운드에서 계속 준비한다.
  - 후보 순위, 점수 손실, 예상 리드, visits, policy prior 같은 상세 텍스트는 화면 하단 응답/디버그 영역과 `Copy Log`로 확인한다.
  - 분석은 대국 AI보다 한 단계 높은 difficulty의 기본 visits/time을 사용한다. 단, 사용자가 이미 더 강한 visits/time을 설정했다면 그 값을 유지한다.
  - 후보 수가 많을 때는 엔진 어댑터가 최소 검색량을 후보 수에 맞춰 내부 상향할 수 있다.
  - 엔진 search나 raw NN policy가 모든 합법점을 채우지 못하면, 나머지 합법점은 `Legal fallback`으로 표시한다. 이 후보에는 점수 손실이나 policy prior가 없을 수 있다.
- `Eval`: 현재 판의 점수 추정과 ownership 정보를 요청한다.
- `New`: 현재 모드로 새 대국을 시작한다.
- `Copy Log`: 현재 런타임 상태, 보드, 수순, 종료 로그, 화면 표시 텍스트, 엔진 진단 정보를 clipboard에 복사한다.

### 후보/착수 평가 색상

- 초록은 좋은 수, 노랑은 약간 아쉬운 수, 빨강은 손실이 큰 수를 뜻한다.
- 회색-파랑 계열은 엔진 점수 손실 정보가 없는 policy 또는 legal fallback 후보를 뜻한다.
- 앱 화면 안의 별도 `Spot legend` 패널은 제거했다. 색상 의미는 이 매뉴얼/도움말 문서로 안내한다.

## Menu 안의 설정

상단 햄버거 버튼을 누르면 아래 설정을 조정할 수 있다. 보조 표시 옵션과 엔진 설정은 기본 플레이 화면의 스크롤 길이를 줄이기 위해 메뉴 안에 둔다. 현재 화면 순서는 `Display menu`, `Engine`이다.

### Engine

- AI 대국에서 엔진 난이도와 검색량을 조정한다.
- 난이도 `- / +`는 `Beginner`, `Casual`, `Club`, `FullAnalysis` 같은 preset을 변경한다.
- `Visits - / +`는 현재 난이도 안에서 엔진 검색 방문 수를 직접 조정한다.
- 값이 커질수록 AI 응수와 분석 품질은 좋아질 수 있지만 응답이 느려질 수 있다.

## Menu 안의 표시 옵션

`Display menu` 영역의 옵션들은 보조 표시 성격이 강한 설정이다.

### Coords

- 바둑판 좌표를 표시한다.
- 현재 구현은 좌측 row label과 하단 column label만 표시한다.
- column label은 바둑 관례에 맞춰 `I`를 건너뛰고 `A B C D E F G H J` 형식으로 표시한다.
- 기본값은 켜짐이다.

### Move nums

- 각 돌 위에 착수 수순 번호를 표시한다.
- 수순이 많아지면 화면 밀도가 높아지므로 복기나 디버그 상황에서 켜는 것이 좋다.

### Last ring

- 마지막 착수 위치를 ring으로 강조한다.
- 꺼져 있을 때도 기존 최소 표시 dot은 유지되어 마지막 수를 완전히 잃지 않도록 한다.
- 기본값은 켜짐이다.

### Engine badge

- 엔진 상태를 작은 badge로 보여준다.
- `Ready`, `Busy`, `Down` 상태를 색상과 텍스트로 구분한다.
- KataGo local process인지 stub fallback인지 빠르게 확인하는 용도다.
- 기본값은 켜짐이다.
- 화면에서는 `Game strip` 아래, 보드 위쪽에 표시된다. `Game strip`이 꺼져 있어도 같은 보드 위쪽 영역에 남는다.

### Ownership

- `Eval` 결과에 포함된 ownership 추정치를 보드 위 heatmap으로 표시한다.
- 흑/백 영향권이 어느 지점에 강하게 형성되어 있는지 보는 보조 지표다.
- `Eval`을 아직 실행하지 않았거나 엔진이 ownership 값을 주지 못하면 켜도 overlay가 표시되지 않을 수 있다.

## 현재 제한사항

- 이 문서는 POC 기준이며, 최종 앱 사용자 매뉴얼은 화면 구조와 용어가 안정된 뒤 다시 정리한다.
- `Ownership`은 엔진 분석 결과에 의존하므로, stub 모드나 분석 실패 상태에서는 의미 있는 값이 없을 수 있다.
- 계가와 사활 판정은 아직 사용자가 죽은 돌을 수동 지정하는 UX가 없어서 KataGo final score와 로컬 면적 추정이 다르게 보일 수 있다.
