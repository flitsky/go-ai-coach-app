## 현재 작업 상태 요약

- 저장소에는 KMP 기반 Android POC skeleton이 생성되어 있다.
- 모듈 구성은 `shared`, `engine-android`, `app-android`이다.
- `shared`에는 9x9/13x13/19x19 보드 크기, 좌표, 착수, 게임 상태, 엔진 DTO, `EngineAdapter` 인터페이스가 있다.
- `engine-android`에는 실제 KataGo 대신 deterministic stub 엔진이 있다.
- `app-android`에는 Compose 기반 9x9 바둑판, 착수 표시, 엔진 응답 표시 영역이 있다.
- 검증 완료 명령은 `:shared:check`, `:app-android:assembleDebug`, `:app-android:testDebugUnitTest`이다.
- 현재는 사람 Black 착수 후 stub AI White가 자동 응수하는 9x9 대국 흐름까지 동작한다.
- 추가 spike 결과, Android `arm64-v8a`용 KataGo Eigen(CPU) 빌드와 앱 UI 연동 smoke test가 성공했다.
- 실제 앱 프로세스에서는 executable을 app private files에서 실행할 수 없고, `nativeLibraryDir/libkatago.so` packaging 경로가 필요하다.

### 성능 조절 구현 메모

- `DifficultyProfile`은 `Beginner`, `Casual`, `Intermediate`, `Strong`, `Full Analysis`를 제공한다.
- 각 profile은 기본 visits/time 값을 가진다.
- 앱 화면의 `Engine` 영역에서 profile과 visits를 `-` / `+` 버튼으로 실시간 조정할 수 있다.
- 조정 시 `EngineAdapter.configure(profile)`이 호출된다.
- stub 엔진은 설정된 profile에 따라 deterministic 착수 우선순위를 일부 다르게 사용한다.
- 실제 KataGo adapter에서는 같은 `configure()` 경계에서 visits/time/model/local-server 모드 등을 반영하면 된다.
- 에뮬레이터에서 `Beginner -> Casual`, `Visits 16 -> 160` 변경이 화면과 엔진 메시지에 반영되는 것을 확인했다.
- 실제 KataGo process adapter에서도 `kata-set-param maxVisits`, `kata-set-param maxTime` 명령이 Android GTP smoke test에서 정상 응답함을 확인했다.
