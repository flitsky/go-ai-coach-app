작성일: 2026-06-08

## 목적

사용자가 “1단계는 추천수로 이기는데 2단계부터 이기기 힘들다”고 보고했다.

이를 검증하기 위해 다음 조건으로 KataGo CLI 시뮬레이션을 수행했다.

- 플레이어: 흑
- 플레이어 착수: 매 차례 KataGo `moveInfos.order == 0` 후보, 즉 최적 추천수
- AI 착수: 백
- AI 선택: 현재 앱의 단계별 랜덤 후보 선택 정책과 동일한 percentile window
- 보드: 9x9
- 룰: Japanese
- komi: 6.5
- 엔진: KataGo v1.16.4, MacBook Metal
- 모델: `kata1-b18c384nbt-s9996604416-d4316597426.bin.gz`
- config: `analysis_learning.cfg`

## 주의

이번 결과는 40수 시점의 중간 형세 평가다.

전체 종국까지 모든 판을 끝낸 최종 승패 검증은 아니다. 다만 “2단계가 왜 갑자기 강하게 느껴지는가”를 판단하기에는 충분한 1차 지표로 볼 수 있다.

## 해석

사용자가 말한 “2단계부터 이기기 힘들다”는 현상은 `빠른 초급 2단계`라면 재현 가능하다.

원인은 `빠른 초급 2단계`의 현재 정책이 너무 강하기 때문이다.

초기 `빠른 초급 2단계`는 “상위 50% 후보 랜덤”이었다. 그런데 B16 분석은 반환 후보 수가 매우 적다. 실제 대국 중 후보가 2~3개만 scoring되는 국면이 많으면 상위 50% pool은 거의 `order 0` 또는 `order 1`만 포함한다.

따라서 “랜덤”이라고 해도 실제로는 다음처럼 동작한다.

- 후보가 2개면: 최적수와 차선 중 랜덤
- 후보가 3개면: 최적수와 차선 중 랜덤
- 후보가 4개면: 최적수~2위 후보 중심

이 구조에서는 2단계가 사실상 “거의 최적수 랜덤”이 되어, 1단계에서 2단계로 넘어갈 때 난이도 점프가 너무 크다.

반대로 `초급 2단계`는 B32 하위 50%를 고르므로, 최적 추천수를 두는 플레이어가 쉽게 앞서는 결과가 나왔다.

## 결론

1. `빠른 초급 2단계`는 현재 너무 강하다.
2. 문제는 엔진이 이상하게 동작해서가 아니라, B16 저후보 환경에서 `상위 50%` 정책이 거의 최적수 선택으로 수렴하기 때문이다.
3. 사용자가 체감한 “2단계부터 어려움”은 합리적인 보고다.
4. `초급 2단계`는 현재 정책 기준으로는 최적 추천수 플레이어가 충분히 이길 수 있는 수준이다.

## 추천 조정안

`빠른 초급`은 “빠른 응답”이 목표이므로 B16은 유지하되, 2단계 선택 정책을 바꾸는 것이 좋다.

1차 추천:

- 빠른 초급 1단계: 현재처럼 하위 50%
- 빠른 초급 2단계: `order 0`을 가능한 한 제외한 차선/중위 후보 랜덤
- 빠른 초급 3단계: 최적수만 착수

구체적으로는 다음 정책이 더 적절하다.

- 후보가 1개뿐이면 그 후보 선택
- 후보가 2개 이상이면 2단계는 `order 1`부터 선택
- 후보가 충분하면 `order 1~상위 60%` 정도에서 랜덤

이렇게 하면 2단계가 “최적수 랜덤”이 아니라 “좋은 후보 중 최적수는 일부러 제외한 랜덤”이 되어, 1단계와 3단계 사이의 학습 곡선이 완만해진다.

## 반영된 조정

2026-06-08 구현에서 `빠른 초급 2단계`는 `ExcludeBestPercentileRange(0, 60)` 정책으로 바뀌었다.

- 후보가 1개뿐이면 그 후보를 선택한다.
- 후보가 2개 이상이면 `order 0`은 제외한다.
- 후보가 충분하면 `order 1`부터 상위 60% 범위에서 랜덤 선택한다.
- 목표는 `FB 1 = 느슨한 하위 후보`, `FB 2 = 좋은 후보지만 최적수 제외`, `FB 3 = 최적수`의 체감 차이를 명확히 만드는 것이다.
